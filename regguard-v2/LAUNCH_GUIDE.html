RegGuard · Publish Guide

Launch on Open VSX

Everything you need to go from source code to installable extension — available to every VS Code fork and Gitpod user on the planet.

⏱ ~2 hours end-to-end

Phase 1 — Prerequisites (15 min)

1
Verify your toolchain Terminal 5 min

You need Node 18+, npm, and Git. Run each check — if any command fails, install the missing tool before continuing.

bash
node --version        # must be ≥ 18.0.0
npm  --version        # must be ≥ 9
git  --version        # any version
💡 If Node is missing, install it from nodejs.org — use the LTS version.
2
Install global publish tools Terminal 2 min

vsce packages your extension into a .vsix file. ovsx uploads it to Open VSX. Both are already in devDependencies but install them globally so you can run them from anywhere.

bash
npm install -g @vscode/vsce ovsx
⚠ On macOS/Linux you may need sudo before this command, or use nvm to avoid permission issues.
3
Unzip and install dependencies Terminal 5 min

Extract the RegGuard zip, enter the directory, and install npm packages — this pulls in LangChain and all dev tools.

bash
unzip regguard.zip
cd regguard
npm install          # installs langchain, @langchain/openai, vsce, ovsx
regguard/ ├── src/ │ ├── agents/ perplexityAgent.ts · grokAgent.ts · claudeAgent.ts │ ├── providers/ diagnosticsProvider.ts · codeActionProvider.ts · sidebarProvider.ts │ ├── extension.ts · scanner.ts · config.ts · types.ts · llm.ts ├── assets/icon.png ← you must add this (see Step 6) ├── package.json · tsconfig.json · README.md

Phase 2 — Open VSX account (20 min)

4
Create an Open VSX account Browser 5 min

Open VSX uses GitHub OAuth — no separate sign-up form.

  • → Go to open-vsx.org
  • → Click Sign in with GitHub (top right)
  • → Authorize the Open VSX OAuth app
  • → You'll be redirected back — you're in
5
Create a publisher namespace and get your token Browser 5 min

Every extension lives under a publisher namespace. Your publisher name goes in package.json and becomes part of the extension ID.

  • → Click your avatar → User Settings
  • → Click Extensions → Create namespace
  • → Enter your publisher name (e.g. regguard or your GitHub username). This must match "publisher" in package.json.
  • → Click your avatar → User Settings → Access Tokens
  • → Click Generate new token — give it a name like regguard-publish
  • → Copy the token immediately — you won't see it again
⚠ Store the token somewhere safe (e.g. your password manager). You'll use it in Step 10.

Phase 3 — Prepare the extension (30 min)

6
Add an icon (required) File 5 min

Open VSX requires a 128×128 PNG icon. Without it, packaging fails.

  • → Create a folder called assets/ inside the regguard directory
  • → Add a 128×128 PNG named icon.png to that folder
  • → Quick option: use any shield icon from icon.kitchen or flaticon.com — resize to 128×128 and save as PNG
bash — confirm it exists
ls assets/icon.png   # should print: assets/icon.png
7
Update publisher in package.json File 2 min

Open package.json and set the publisher field to exactly the namespace you created in Step 5. Also update the repository.url to your GitHub repo.

package.json — lines to change
"publisher": "YOUR_NAMESPACE_FROM_STEP_5",
"repository": {
  "type": "git",
  "url":  "https://github.com/YOUR_GITHUB_USERNAME/regguard"
},
ℹ Open VSX requires a repository field and a valid license field — both are already in the template. Just update the URL.
8
Push to GitHub (required for Open VSX) Terminal 5 min

Open VSX requires extensions to have a public source repository. Create a GitHub repo and push.

bash
# Create a new repo on github.com first, then:
git init
git add .
git commit -m "feat: initial RegGuard release"
git remote add origin https://github.com/YOUR_USERNAME/regguard.git
git branch -M main
git push -u origin main
9
Compile and package the .vsix Terminal 3 min

TypeScript → JavaScript, then bundle everything into a single .vsix file that Open VSX will accept.

bash
npm run compile    # compiles TypeScript to out/
vsce package        # creates regguard-0.1.0.vsix in the current folder
💡 If vsce package warns about missing fields, it will tell you exactly which ones. Fix them in package.json and re-run.

Verify the package is healthy:

bash
vsce ls             # lists all files that will be included — check for bloat
ls -lh *.vsix       # should be a few MB max

Phase 4 — Publish (15 min)

10
Publish to Open VSX Terminal 3 min

Use the ovsx CLI to upload the .vsix. Pass your token via the -p flag (or set the env variable to avoid it appearing in shell history).

Option A — env variable (recommended):

bash
export OVSX_TOKEN=your_token_from_step_5
ovsx publish regguard-0.1.0.vsix -p $OVSX_TOKEN

Option B — npm script (already wired up):

bash
export OVSX_TOKEN=your_token_from_step_5
npm run publish:ovsx   # runs: ovsx publish *.vsix -p $OVSX_TOKEN

Expected output:

success
Successfully published regguard@0.1.0 to Open VSX!
11
Verify it's live Browser 2 min

Your extension should appear within a few minutes.

  • → Go to open-vsx.org and search RegGuard
  • → Or go directly to open-vsx.org/extension/YOUR_PUBLISHER/regguard
  • → In VS Code (or VS Codium), open Extensions, search RegGuard, click Install
💡 Open VSX also serves the VS Code Marketplace API fallback for VS Codium, Gitpod, GitHub Codespaces, and Eclipse Theia — one publish, every editor.

Phase 5 — Updates (ongoing)

12
Publishing future versions Terminal 5 min per update

Bump the version in package.json, then recompile and republish. Semantic versioning: patch = bug fix, minor = new feature, major = breaking change.

bash — patch release workflow
# 1. Bump version (edits package.json and creates a git tag)
npm version patch   # 0.1.0 → 0.1.1

# 2. Compile + package
npm run compile
vsce package

# 3. Publish
npm run publish:ovsx

# 4. Push git tag
git push --follow-tags
13
Automate with GitHub Actions (optional) File 10 min

Add this workflow file — it publishes automatically whenever you push a version tag.

.github/workflows/publish.yml
name: Publish to Open VSX

on:
  push:
    tags:
      - 'v*'   # triggers on v0.1.0, v1.0.0, etc.

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run compile
      - run: npx vsce package
      - run: npx ovsx publish *.vsix -p $OVSX_TOKEN
        env:
          OVSX_TOKEN: ${{ secrets.OVSX_TOKEN }}
💡 Add OVSX_TOKEN in your GitHub repo → Settings → Secrets → Actions → New repository secret.

Troubleshooting

?
  • ! "Publisher not found" — the namespace in Step 5 must exactly match "publisher" in package.json (case-sensitive)
  • ! "Missing icon" — assets/icon.png must exist and be a valid 128×128 PNG before running vsce package
  • ! "Repository field required" — add a repository object to package.json pointing to your public GitHub repo
  • ! Token rejected — tokens expire; generate a new one in Open VSX user settings
  • ! TypeScript compile errors — run npm install first to ensure LangChain types are present, then npm run compile
  • ! Extension not appearing in search — Open VSX indexes within ~5 minutes; search by exact name RegGuard
